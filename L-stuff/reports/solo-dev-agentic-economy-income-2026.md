## The Moat Question: What Agents Cannot Commoditize

### 1. Taste and Judgment in Ambiguous Domains
Agents are terrible at deciding WHAT to build. Excellent at building what you tell them. The person who identifies the right problem, frames it correctly, evaluates whether the solution works retains enormous value.

### 2. Integration with Messy Legacy Systems
Agents cannot navigate undocumented APIs, inconsistent data formats, or systems requiring phone calls to vendor support. Explicitly the biggest enterprise adoption bottleneck.

### 3. Trust and Relationships
Small business owners don't buy from agents. They buy from people they trust. Relationship IS the moat.

### 4. Multi-System Orchestration Under Uncertainty
Agents execute defined workflows. They cannot handle "API is down, fallback has different schema, client changed requirements mid-build" reality.

### 5. Security and Adversarial Thinking
Tools for multi-agent AI ahead of tools for SECURING multi-agent AI. Smart contract auditing, agent security review, prompt injection testing — adversarial thinking remains human advantage.

### 6. When Everyone Has an Agent
Agents converge on same answers (same sources). Moat is in:
- **Proprietary data/access** (relationships, niche expertise)
- **Execution speed** (shipping this week while others plan)
- **Compounding reputation** (5 great reviews > 500 agent-generated proposals)
- **System-level thinking** (agents optimize locally; humans seeing full system win)

## Specific Opportunities in Agentic Infrastructure

### 1. Agent-to-Agent Communication Middleware
50% of agents operate in isolation. Practical connectors for MCP protocol needed.

### 2. Deterministic Guardrails Outside LLM Loop
Enterprise unlock. Cedar-based policy services, NeMo Guardrails point direction. Developer-friendly guardrails library for Claude Code agents has no good solution.

### 3. Agent Evaluation for Small Teams
Existing eval tools (Langfuse, Arize, Maxim) target enterprise pricing. No lightweight CLI-first eval for indie developers.

### 4. Agent Observability Dashboard
89% of orgs have some observability but tools fragmented. Unified simple dashboard for monitoring multiple agents across providers — no dominant solution for small teams.

### 5. MCP Server Testing Framework
17,867 MCP servers on mcp.so. No standardized way to test, verify, or assess security. Testing/certification service is a gap.
